HTML

  • !DOCTYPE html: Declares the document type and version of HTML being used
  • html: Wraps all other HTML content on the page
  • head: Contains information about the webpage
  • meta: Provides metadata about the HTML document, such as character encoding and viewport settings
  • title: Sets the title of the webpage, displaying it in the users browser tab
  • body: Represents the visible content shown to the user
  • h1 to h6: Headings for organising content, with "h1" being the biggest and "h6" being the smallest
  • p: Defines a paragraph of text
  • a: Creates hyperlinks within or out of the webpage
  • img: Embeds images on the page
  • ul: Defines an unorederd (bulleted) list
  • ol: Defines an ordered (numbered) list
  • div: A generic container for grouping and styling content
  • span: A generic inline container often used for styling specific text
  • input: Generates various types of input fields
  • button: Creates a clickable button
  • br: Inserts a break line within the text content, as shown above

Git

  • git init: Initializes a new Git repository in the current directory.
  • git clone [URL]: Copies a remote repository to your local machine.
  • git add [file]: Stages a file for commit.
  • git commit -m "[message]": Records staged changes with a descriptive message.
  • git status: Displays the current status of your repository.
  • git log: Shows a history of commits along with their details.
  • git pull: Fetches and merges remote changes into your local branch.
  • git push: Sends your local commits to a remote repository.
  • git branch: Lists existing branches and indicates the current branch.
  • git checkout [branch/commit]: Switches to a different branch or commit.
  • git merge [branch]: Integrates changes from one branch into another.
  • git remote -v: Displays remote repositories and their URLs.
  • git fetch: Retrieves changes from remote repositories without merging.
  • git reset [file]: Unstages a file, undoing a previous git add.
  • git diff: Shows the differences between your working directory and the last commit.
  • git stash: Temporarily shelves changes for later use.
  • git tag [name]: Creates a tag to mark a specific commit (like a release).
  • git remote add [name] [URL]: Adds a new remote repository.
  • git config: Manages configuration settings for Git.
  • git commit --amend: Edits the last commit's message or content.
  • git rebase [branch]: Applies changes from one branch onto another.
  • git log --oneline: Displays a condensed version of the commit log.
  • git push --force: Overwrites remote history with your local changes.
  • .gitignore: A file that lists patterns of files to be ignored by Git.
  • git revert [commit]: Creates a new commit that undoes changes from a previous commit.
